# Ace Framing Skill

토픽 프레이밍을 정교화하고 자가검토를 수행한다.

## 발동 조건 (OR)

1. Master가 `/ace-framing` 직접 호출
2. Ace가 토픽 프레이밍 시 아래 중 하나에 해당한다고 판단:
   - 토픽 제목/설명만으로 목적·제약·성공기준 중 2개 이상 불명확
   - 범위가 2개 이상 독립 서브시스템을 포함할 가능성
   - 이전 유사 토픽에서 프레이밍 수정이 발생했던 패턴 (`ace_memory` 참조)

## 1. 정교화 질문 프로토콜

Master에게 질문하여 프레이밍을 구체화한다.

**규칙:**
- 한 번에 한 질문만. 여러 질문을 묶지 않는다.
- 객관식(A/B/C 선택지) 우선. 열린 질문은 선택지를 줄 수 없을 때만.
- 집중 축: **목적** (왜 하는가) / **제약** (무엇을 피해야 하는가) / **성공기준** (완료 기준이 무엇인가)
- 범위 초과 감지 시 질문 전에 먼저 분해 플래그를 세운다:
  > "이 토픽은 [A], [B] 두 개의 독립 서브시스템을 포함합니다. 먼저 분해하겠습니까, 아니면 통합으로 진행하겠습니까?"

## 2. 추천 근거 필수화

Ace가 결정축에 대해 추천을 제시할 때 반드시 아래 구조를 따른다:

```
추천: [선택지]
이유: [왜 이것이 맞는가 — 구체 근거]
기각 이유: [나머지 선택지를 왜 선택하지 않는가]
```

"권고합니다" 한 줄로 끝내는 것은 금지. 근거 없는 추천은 추천이 아니다.

## 3. 프레이밍 자가검토

Ace 1차 발언 직후, 내부적으로 아래 4항목을 점검한다.
**걸린 항목이 있을 때만 발언에 표시한다. "4개 통과" 보고는 금지.**

| 항목 | 점검 내용 |
|---|---|
| 전제 누락 스캔 | 미확인 전제, 미정의 용어, 불완전한 결정축이 있는가? |
| 결정축 간 모순 | 축끼리 충돌하거나 분석과 추천이 불일치하는가? |
| 범위 적정성 | 단일 토픽으로 처리 가능한 크기인가? |
| 모호성 점검 | 이중 해석이 가능한 표현이 있는가? |

걸린 항목은 발언 내에서 즉시 수정하고 수정 사실을 한 줄로 표시한다.
