#### 2.1.7 연구원 프로토콜 검토 요청사항
- **출처**: /pages/5046992907

**수집 내용:**

**1. 수익성 검토**
- 일정 금액 이상 또는 고복잡도 제안 건에 대해 하이브리드러닝 운영팀 사전 리뷰 게이트 적용
- 레이트카드는 "about 단가" 형태라도 상·중·하 밴드(Price Band)로 기준선 고정
- 핵심 목적: 강사 개별 확인 전에도 1차 제안(Indicative Quote) 가능하게 만드는 것
- 효과: 견적 편차 축소 → 마진 방어 → 고객 협상 기준점 확보
- 운영 룰: "밴드 기반 1차 제안 가능"을 공식 원칙으로 선언(단, 예외 조건은 게이트로 관리)

**2. 강사풀 확대**
- 비용 효율형 강사(저가 라인) 병행 확보 구조 반영보강(풀 협소 → 단가 경쟁력 저하 원인, 일정 사전 점검 risk 해소)
- 산업/유형별로 "필수 커버리지 맵"을 만들고 공백 영역 중심으로 소싱 우선순위 설정(제조/금융/코칭/AI/법정 등)
- 가용성 리스크 방지: 핵심 과정은 대체 강사(Backup) 2~3명까지 확보하는 운영 원칙 적용

**3. 강사 품질 사전 검토**
- 방향: 강사 소싱→평가→등록→운영으로 품질관리 체계를 상향
- 강사 등록(온보딩) 단계에서 평가검증 방식: 시범 강의(샘플) + 평가 루브릭(항목/배점) + 최소 승인 라인
- 등급 부여: 프리미엄/스탠다드/비용 효율형 라인으로 초기 등급 확정
- 운영 단계 품질 관리 성과지표: 만족도, 재의뢰율, 클레임/리워크 발생률
- 리스크 제어: 기준 미달 시 등급 조정/재교육/풀 제외(Exit rule) 명문화
- 단가/가용성도 등록 단계에서 사전 합의, 등록 시 밴드/조건을 사전 합의

**4. 법정교육 등 기본 상품(사전 준비형 운영)**
- 법정교육: 표준 패키지(콘텐츠·외국어 버전)을 사전 보유
- 안전보건교육: 산업별 6종 연간 반복 수요 전제로 운영
- 연간 운영 캘린더(업데이트 시점) + 버전관리(개정 이력/적용 범위) 체계화

**5. 이러닝 개발 프로토콜(검토 기간 단축 + 리드타임 레인 분리 + Fast-track)**
- 개발 검토 기간(현 2주 수준)에 대해 단축 방안 검토
- 승인 라인 슬림화(필수 승인만), 체크리스트 표준화, 병렬 검토, 템플릿/재사용 확대
- 리드타임 단축(레인 분리): 경미 수정 / 부분 개편 / 신규 개발
- Fast-track 검토 적용 필수 조건: 기존 템플릿 기반 재사용 가능(30% 이상), 범위 제한(모듈 단위/부분 수정)
- 남용 방지: 동시 진행 WIP 상한(2~3건) 설정
