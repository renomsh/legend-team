# Fin — 세션 전환 적정 타이밍 기준

## 판단 축

컨텍스트 비용 vs 토픽 응집도.

---

## 세션 유지가 유리한 구간

- 같은 토픽의 연속 작업 (설계 → 구현 → 검증)
- 직전 발언 내용을 역할이 참조해야 할 때
- 세션 오픈 비용(~900 토큰 fast-path 로드)이 아깝다고 판단될 만큼 짧은 작업

## 새 세션이 유리한 시점

- 토픽이 명확히 전환될 때 (DB매핑 → 역할 재정의처럼 맥락 단절)
- 컨텍스트가 쌓여 compaction 발생 이후 — 이미 손실 발생
- 역할 메모리를 새로 선택 로드해야 할 타입 변경 시

---

## 적정 타이밍 기준

| 조건 | 권장 |
|------|------|
| 토픽 동일 + 작업 30분 이내 | 유지 |
| 토픽 전환 | 새 세션 |
| compaction 1회 발생 | 즉시 닫고 새 세션 |
| 역할 발언 누적 5회 이상 + 다음 작업이 다른 타입 | 새 세션 |

**핵심:** compaction 발생 = 강제 세션 종료 신호.
compaction 이후 유지는 손실 구간에서 작업하는 것과 같음.
